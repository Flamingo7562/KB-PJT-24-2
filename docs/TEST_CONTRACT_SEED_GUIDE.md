# 테스트 근로계약·에스크로 Seed 사용법

이 Seed는 이슈 #17의 합성 데이터만 생성하거나 초기 상태로 되돌립니다. 실제 개인정보·계좌정보는 포함하지 않습니다. 사장님과 근로자 계정, 사업장, 근로계약, 지갑 원장, 300,000원 에스크로와 대기 중 정산을 한 번에 준비합니다.

## 실행

Docker Desktop을 실행한 뒤 저장소 루트에서 다음 한 줄만 실행합니다.

```powershell
npm run db:seed:contract
```

명령은 먼저 미적용 Flyway migration을 적용하고, MySQL이 정상일 때 Seed를 실행합니다. 같은 명령을 다시 실행하면 전용 테스트 계정과 `[TEST-17]` 근무 건을 아래 초기 상태로 되돌립니다. 다른 사용자의 데이터는 삭제하지 않습니다.

| 항목 | 테스트 값 |
| --- | --- |
| 사장님 로그인 | `test_owner_17` / `Test1234!` |
| 근로자 로그인 | `test_worker_17` / `Test1234!` |
| 근무 | 2026-08-01 09:00~18:00, 무급 휴게 60분 |
| 일급 | 300,000원 |
| 사장님 지갑 | 가용 700,000원 + 잠금 300,000원 |
| 근로자 지갑 | 가용 0원, 에스크로 확보액 300,000원 |
| 상태 | 근무 `ACCEPTED`, 에스크로 `HELD`, 정산 `WAITING` |

Seed SQL은 `backend/src/test/resources/db/seed/test-contract-escrow.sql`에 있습니다. 전용 계정은 다른 테스트 데이터와 섞어 쓰지 마세요. 재실행하면 두 계정의 지갑 원장·Mock 이체와 해당 근무 건의 근태·분쟁·문서·정산 데이터가 초기화됩니다.

## 확인할 흐름

1. 사장님 홈에는 총액이 아니라 **가용 잔액 700,000원**을 표시하고, 300,000원은 잠금액으로 분리합니다.
2. 근로자에게 300,000원은 확보액으로 보여주되 지갑 가용 잔액이나 출금 가능액에는 더하지 않습니다.
3. 정상 퇴근 시 정산을 `SCHEDULED`로 바꾸고 `due_at = 퇴근 시각 + 24시간`으로 저장합니다.
4. 유예 중 사장님 승인 또는 기한 만료 자동 지급은 같은 정산 트랜잭션을 사용합니다. 성공하면 에스크로 `RELEASED`, 사장님 잠금액 0원, 근로자 가용 잔액 300,000원이 되어야 합니다.
5. 이의 제기 시 에스크로와 정산을 `ON_HOLD`로 바꾸고 자동 지급 대상에서 제외합니다.

테스트를 처음부터 다시 하려면 Seed 명령을 재실행하면 됩니다. 전체 DB를 지우는 `docker compose down -v`나 Flyway `clean`은 사용하지 않습니다.
